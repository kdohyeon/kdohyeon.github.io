홈 화면 CMS 도입으로 콘텐츠 유형 2개 → 6개 확장

Situation

트렌비 홈 화면은 프론트엔드 하드코딩 방식으로 관리되고 있었습니다. 새로운 콘텐츠 유형(타임딜, 이미지 배너 등)을 추가하거나 노출 순서를 변경하려면 매번 개발자가 직접 코드를 수정해야 했고, 확장성과 유지보수성이 모두 낮은 상태였습니다.

Behavior

JSON 타입 기반 유연한 콘텐츠 설계

상품·이미지·타임딜 등 다양한 콘텐츠 유형을 하나의 구조로 처리하기 위해, 콘텐츠 데이터를 MySQL JSON 타입으로 저장하고 @JsonType으로 역직렬화하는 방식을 설계했습니다. 이 접근 덕분에 새로운 콘텐츠 유형 추가 시 테이블 스키마를 변경하지 않아도 됩니다.

인력 공백 대응: '오전 소통, 오후 집중 개발' 루틴

협업 백엔드 개발자가 신혼여행으로 개발 공백이 발생했습니다. 제한된 시간 내 혼자서 소통과 메인 개발을 모두 소화하기 위해 오전 소통·협의, 오후 집중 개발이라는 전략적 루틴을 세워 일정을 지킬 수 있었습니다.

팀 커뮤니케이션 개선

백엔드-프론트엔드 전용 슬랙 채널을 별도로 운영하여 대기 시간을 최소화하고 협업 효율을 높였습니다.

Impact

  • 콘텐츠 유형 확장: CMS 도입 후 처리 가능한 홈 화면 콘텐츠 유형이 2개 → 6개로 확장.
  • 렌더링 성능 향상:
    • Speed Index: 5.0s → 3.5s (30% 개선)
    • Total Blocking Time: 0.8s → 0.6s (25% 개선)
    • LCP: 6.8s → 6.3s (7% 개선)
  • 교훈 전파: 단일 키 캐싱으로 인한 타임아웃 문제를 이후 스타일 서비스 프로젝트에서 개별 단위 캐싱으로 개선하는 데 활용.
