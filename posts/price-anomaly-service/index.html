역량 · 서비스 개발

다이나믹 프라이싱 오류 탐지 시스템 구축 — Z-Score 알고리즘 도입과 특허 등록

2020 쿠팡

Situation

쿠팡은 타 플랫폼과 가격을 자동으로 맞추는 다이나믹 프라이싱(Dynamic Pricing) 정책을 운영합니다. 이 과정에서 판매자의 실수나 시스템 오류로 '가짜 낮은 가격'이 등록되는 문제가 발생했습니다.

  • 예시: 3개 묶음 상품을 1개 가격으로 잘못 등록 → 다이나믹 프라이싱이 해당 가격을 기준으로 따라내려 회사 경제적 손실 발생
  • 기존에는 이런 이상 가격을 담당자가 수동으로 모니터링해야 했으며, 조기 발견 및 즉각 대응이 어려운 구조였습니다.
  • 개발 기간: 2020. 01 ~ 2020. 03 (3개월)
  • 개발 인원: 백엔드 3명
  • 담당 역할: 이상 탐지 알고리즘 방향 제안 (기여도 100%), 알고리즘 구현 (기여도 30%), 서비스 개발 참여
  • 기술 스택: Java, Spring Boot, Redis, Couchbase, Kafka

Behavior

수집 → 분석 → 리포팅 3단계 파이프라인으로 시스템을 설계하고 구현했습니다.

  • 데이터 수집: 수백만 건의 주문·가격 데이터를 Kafka를 통해 실시간 수집. Redis(5분 단위)와 Couchbase 검색 엔진으로 시간대별·카테고리별 그룹핑 처리.
  • 이상 탐지 알고리즘 설계 (제안 기여도 100%):
    • 원가 대비 과도한 가격 변동, 마진율 급락, 비정상적인 판매량 급증을 '이상'으로 정의.
    • Z-Score 통계 기법을 활용하여 데이터 분포에서 벗어난 이상치를 탐지.
    • 정상 가격 범위(Threshold)를 동적으로 설정하여 탐지 정확도를 높임.
  • 리포팅 시스템: 분석 결과를 백오피스 대시보드와 자동 이메일 발송으로 유관 부서에 즉시 공유.

Impact

  • 수동 모니터링 시스템화: 담당자가 수동으로 확인하던 이상 가격 탐지를 시스템이 자동으로 처리하여 운영 효율성 증대 및 경제적 리스크 감소.
  • 즉각 대응 환경 구축: 이상 가격 발생 시 실시간 알럿으로 유관 부서가 빠르게 인지하고 대응 가능.
  • 특허 등록: 이 프로젝트에서 개발한 이상 탐지 알고리즘을 기반으로 특허가 등록되었습니다.
