TPM · 프로세스 최적화

파이프라인 생성 공수 80% 절감 — 데이터 기반 개선 제안부터 구현까지

2025 카카오뱅크

Situation

금융 도메인 특성상 모든 마케팅 메시지는 엄격한 결재 프로세스를 거치며, 보안·추적성을 위해 1회성 파이프라인 구조로 운영됩니다. 이로 인해 다음과 같은 운영 비효율이 발생했습니다.

  • 반복 작업 과다: 연간 생성 파이프라인의 약 50%가 동일한 템플릿·속도·필터를 사용함에도 매번 수동으로 신규 생성해야 했습니다. (예: 오전/오후 분할 발송, 채널별 분할 발송 등)
  • 높은 운영 리소스: 파이프라인 1개 생성에 최소 5~10분의 수동 입력 시간이 소요되어 생산성 저하 및 휴먼 에러 가능성이 존재했습니다.

Behavior

단순 기능 구현에 머물지 않고, 데이터 기반의 문제 제기부터 사용자 중심 설계까지 직접 주도했습니다.

  • 데이터 기반 개선 제안: 1년간의 파이프라인 생성 로그를 분석하여 동일 패턴 비중(약 50%)을 산출. 이를 근거로 '파이프라인 복사 기능' 도입 효과(ROI)를 정량화하여 유관 부서에 제안.
  • 요구사항 기획 및 명세: 직접 기획서를 작성하고 사용자 인터뷰를 통해 워크플로우를 파악. 금융 보안 정책상 '반드시 새로 입력해야 하는 필드'와 '자동 복사 가능한 필드'를 구분하여 정책 수립.
  • 비즈니스 로직 구현: 결재 프로세스 정합성을 유지하면서 중복 입력을 최소화하는 복사 로직 설계 및 구현.

Impact

  • 파이프라인 생성 공수 80% 절감: 평균 생성 소요 시간 7.5분 → 1.5분으로 단축.
  • 캠페인 대응 속도 향상: 대규모 마케팅 캠페인 시 파이프라인 분할 발송 대응이 비약적으로 빨라짐.
  • 핵심 기능으로 정착: 릴리즈 이후 마케팅 운영팀의 필수 기능으로 자리잡아 안정적으로 운영 중.
